🚀 Guía de Pruebas con Postman - The Destiny Vault API

📦 IMPORTAR COLECCIÓN:
  Archivo JSON importable: backend/postman/TheDestinyVault.postman_collection.json
  En Postman: File → Import → Selecciona el archivo .json

URL Base: http://localhost:5000/api

💡 Tips:
  1. Postman gestiona las cookies httpOnly automáticamente (no necesitas copiar tokens).
  2. Primero haz Login para obtener la cookie, luego prueba las rutas protegidas.
  3. Para subir imágenes, selecciona "form-data" en el Body y cambia el tipo del campo "image" a "File".

---

## 1. Autenticación (Auth)

🔹 POST /api/auth/register — Registrar usuario
Body (JSON):
{
  "nombre": "Test User",
  "email": "test@ejemplo.com",
  "password": "password123",
  "telefono": "+506 12345678",
  "ubicacion": "San José",
  "direccion_precisa": "Barrio Escalante"
}

🔹 POST /api/auth/login — Iniciar sesión
Body (JSON):
{
  "email": "yosimarvv@gmail.com",
  "password": "yosi12345."
}

🔹 POST /api/auth/logout — Cerrar sesión (limpia cookie JWT)

🔹 GET /api/auth/me — Perfil del usuario autenticado (requiere cookie)

🔹 POST /api/auth/check-email — Verificar si un email existe
Body: { "email": "yosimarvv@gmail.com" }

🔹 POST /api/auth/reset-password — Restablecer contraseña
Body: { "userId": "ID_DEL_USUARIO", "newPassword": "nuevaPassword123" }

## 2. Vehículos — CRUD + Upload de Imágenes (Multer)

🔹 GET /api/vehicles — Listar todos (público, con paginación)
  Query params: page, limit, search, type, fuel, transmission, tag, color, minPrice, maxPrice, minYear, maxYear, sort, order

🔹 GET /api/vehicles/:id — Ver un auto específico

🔹 POST /api/vehicles — Crear auto (Admin) ⭐ Soporta imagen
  Opción A - JSON: Body → raw → JSON con campo "image" como URL de texto
  Opción B - Multer: Body → form-data:
    name (text): "Honda CR-V"
    price (text): "32000"
    year (text): "2024"
    image (file): [seleccionar archivo]

🔹 PUT /api/vehicles/:id — Actualizar auto completo (Admin, soporta imagen)
🔹 PATCH /api/vehicles/:id — Actualizar campo específico (Admin, soporta imagen)
🔹 DELETE /api/vehicles/:id — Eliminar auto (Admin, también elimina la imagen del disco)

## Códigos HTTP de Respuesta

  200 — Operación exitosa
  201 — Recurso creado exitosamente
  400 — Datos inválidos o archivo no permitido
  401 — No autenticado / credenciales inválidas
  403 — Acceso denegado (se requiere rol admin)
  404 — Recurso no encontrado
  429 — Demasiadas peticiones (rate limit)
  500 — Error interno del servidor
