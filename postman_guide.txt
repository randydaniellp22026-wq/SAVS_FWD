🚀 Guía de Pruebas con Postman - The Destiny Vault API

Esta guía te permitirá probar todos los endpoints de tu nuevo backend en MySQL, cubriendo el CRUD completo (Create, Read, Update, Delete) incluyendo el método PATCH. Asegúrate de que el servidor esté corriendo (`npm run dev:full`).

URL Base: http://localhost:5000/api

---

## 1. Autenticación (Auth)
Para probar las rutas protegidas, primero debes iniciar sesión para obtener una cookie de autenticación.

🔹 Login (Iniciar Sesión)
- Método: POST
- URL: http://localhost:5000/api/auth/login
- Body (JSON):
{
  "email": "yosimarvv@gmail.com",
  "password": "yosi12345."
}

🔹 Perfil Actual (Ver mis datos)
- Método: GET
- URL: http://localhost:5000/api/auth/me

---

## 2. Vehículos (Inventario) - CRUD COMPLETO
Puedes gestionar los autos usando estos endpoints:

🔹 1. Crear un Auto (Create)
- Método: POST
- URL: http://localhost:5000/api/vehicles
- Body (JSON):
{
  "brand": "Toyota",
  "model": "Corolla",
  "year": 2024,
  "price": 25000,
  "mileage": 0,
  "color": "Blanco",
  "status": "Disponible",
  "image_url": "https://example.com/corolla.jpg"
}

🔹 2. Listar todos los autos (Read)
- Método: GET
- URL: http://localhost:5000/api/vehicles

🔹 3. Ver un auto específico (Read)
- Método: GET
- URL: http://localhost:5000/api/vehicles/1

🔹 4. Actualizar un auto completo (Update)
- Método: PUT
- URL: http://localhost:5000/api/vehicles/1
- Body (JSON):
{
  "brand": "Toyota",
  "model": "Corolla XSE",
  "year": 2025,
  "price": 28000,
  "mileage": 10,
  "color": "Rojo",
  "status": "Vendido",
  "image_url": "https://example.com/corolla.jpg"
}

🔹 5. Actualizar un campo específico de un auto (Patch)
- Método: PATCH
- URL: http://localhost:5000/api/vehicles/1
- Body (JSON):
{
  "price": 26000
}

🔹 6. Eliminar un auto (Delete)
- Método: DELETE
- URL: http://localhost:5000/api/vehicles/1

---

## 3. Usuarios (Gestión) - CRUD COMPLETO (Solo Admin)
Requiere haber iniciado sesión como administrador.

🔹 1. Crear un Usuario (Create)
- Método: POST
- URL: http://localhost:5000/api/users
- Body (JSON):
{
  "name": "Nuevo Usuario",
  "email": "nuevo@ejemplo.com",
  "password": "password123",
  "role": "user"
}

🔹 2. Listar todos los usuarios (Read)
- Método: GET
- URL: http://localhost:5000/api/users

🔹 3. Ver un usuario específico (Read)
- Método: GET
- URL: http://localhost:5000/api/users/2

🔹 4. Actualizar un usuario completo (Update)
- Método: PUT
- URL: http://localhost:5000/api/users/2
- Body (JSON):
{
  "name": "Usuario Modificado",
  "email": "modificado@ejemplo.com",
  "role": "admin"
}

🔹 5. Actualizar estado/rol de un usuario (Patch)
- Método: PATCH
- URL: http://localhost:5000/api/users/2
- Body (JSON):
{
  "role": "admin"
}

🔹 6. Eliminar un usuario (Delete)
- Método: DELETE
- URL: http://localhost:5000/api/users/2

## 4. IA Chatbot
Prueba cómo la IA utiliza tu base de datos de MySQL para responder.

🔹 Preguntar al Chatbot
- Método: POST
- URL: http://localhost:5000/api/chatbot
- Body (JSON):
{
  "message": "¿Qué autos tienes disponibles en color blanco?"
}

## 5. Otros Endpoints Útiles

- Settings (GET): http://localhost:5000/api/settings

---

💡 Tips para Postman:
1. Headers: Postman suele manejar el Content-Type: application/json automáticamente cuando eliges "Raw" -> "JSON" en el Body.
2. Cookies: Como tu sistema usa httpOnly cookies para el JWT, Postman las gestionará solo. No necesitas copiar y pegar tokens manualmente para los métodos de usuarios.
3. Logs: Revisa la terminal de tu backend para ver las consultas SQL generadas.
